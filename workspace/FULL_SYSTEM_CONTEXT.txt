# Security Audit Summary (Generated by CyberClaw via Gemini CLI)
## Findings:
- **Critical Risk:** `Control UI allowInsecureAuth: true`. This setting allows administrative access without proper security validation.
- **Data Sprawl:** Numerous backup files (`openclaw.json.bak.*`) may contain sensitive legacy keys.
- **Integrity Risk:** `exec-approvals.json` is a prime target for RCE injection.

## Recommendations:
1. Set `allowInsecureAuth: false`.
2. Securely delete old `.bak` files.
3. Move `workspace/` outside of `.openclaw/` to prevent path traversal risks.
